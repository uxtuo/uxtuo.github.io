Spark实践

发表于 2018-02-02 | 分类于 Bigdata

简介

Apache Spark 是专为大规模数据处理而设计的快速通用的计算引擎,一个用来实现快速而通用的集群计算的平台,扩展了广泛使用的MapReduce计算模型，能高效支持更多的计算模式，包括交互式查询和流处理。在处理大规模数据集的时候有优势，Spark的一个重要特点就是能够在内存中计算，因而比MapReduce更快，即使在磁盘上进行的复杂计算，Spark依然更加高效。

运行环境配置

请务必关闭防火墙，包括iptables及selinux

java

三台虚拟机均需配置java环境

$tar xvf jdk-7u25-linux-x64.tar.gz -C /usr/local/ 
$ vi /root/.bash_profile
$ source /root/.bash_profile

export PATH
export JAVA_HOME=/usr/local/jdk1.7.0_25
export JRE_HOME=$JAVA_HOME/jre
export CLASSPATH=$JAVA_HOME/lib:$JRE_HOME/lib
export PATH=$JAVA_HOME/bin:/usr/local/hadoop-2.7.2/bin:/usr/local/hadoop-2.7.2/sbin:$PATH
export HADOOP_COMMON_LIB_NATIVE_DIR=$HADOOP_HOME/lib/native
export HADOOP_OPTS="-Djava.library.path=$HADOOP_HOME/lib"

ssh

在三台虚拟机上执行命令生成密钥

# ssh-keygen

ssh将各个机器的密钥拿来238上面合成authorized_keys，然后分发到各个节点

$ ssh root@192.168.4.237 cat /root/.ssh/id_rsa.pub >> /root/.ssh/authorized_keys

$ ssh root@192.168.4.50 cat /root/.ssh/id_rsa.pub >> /root/.ssh/authorized_keys

$ scp authorized_keys root@192.168.4.237:/root/.ssh/

$ scp authorized_keys root@192.168.4.50:/root/.ssh/

修改配置文件

文件夹需要在三个虚拟机都创建，配置文件可以在一台机器修改，然后rsync同步到其余两台机器

mkdir -p {tmp,hdfs/{data,name}}

tmp用来存储临时生成的文件

hdfs用来存储集群数据

hdfs/data用来存储真正的数据

hdfs/name用来存储文件系统元数据

遇到的坑

1.请务必关闭防火墙，包括iptables和selinux，否则会导致集群无法通信，页面无法访问等问题

2.启动后有警告Unable to load native-hadoop library for your platform... using builtin-java classes where applicable，是因为Apache提供的hadoop本地库是32位的，而在64位的服务器上就会有问题，因此需要自己编译64位的版本，解决方案是 找个包，解压到目录，修改环境变量，操作后问题解决，原作者环境跟我的环境不一样，不确定是不是所有人都会碰到这个问题

$tar -xvf hadoop-native-64-2.7.0.tar -C /~hadoop-2.7.2/lib/native 

$ tar -xvf hadoop-native-64-2.7.0.tar -C ~/hadoop-2.7.2/lib

3.第一次启动后进程都正常，页面无法访问，查看监听状态发现监听到了ipv6的端口，hadoop的配置文件明明标注的是监听到ipv4的端口，解决方案如下。

vi /etc/sysctl.conf
添加以下内容并执行stop-all.sh和start-all.sh重启hadoop服务
#disable ipv6
net.ipv6.conf.all.disable_ipv6 = 1
net.ipv6.conf.default.disable_ipv6 = 1
net.ipv6.conf.lo.disable_ipv6 = 1
